Expert Guidance

AI Vision is an Iterative Process

We sought advice from Joshua, CEO of Amherst Intelligence Security, whom we were referred to by Igor. He gave us some general advice on building a machine-vision system from scratch.

01

Don't point into the bin

A messy pile of wet containers is a dead end. Focus on the entrance — the throat — where the scene can be controlled.

02

More cameras beats better cameras

Multiple cameras are necessary, even if they're inexpensive. Collecting data from more angles outperforms a single high-end unit by reducing false negatives & positives

03

You need a lot of data

This is a scaling problem. The model needs thousands of real return events in real-world variation before it's useful.

04

Expect a long iteration cycle

Even with the right approach, getting something that works even a little in a real food-court environment takes significant time and tuning.

Design principle from Joshua: Watch the passage, not the pile. The throat concept on the next slide was built on this advice.

07 / 11
Design Constraint

The Modularity Challenge

USEFULL Return Station Concept

14" slot, controlled geometry, designed for this system

  • This concept was designed for USEFULL's new return station — a controlled, known geometry.
  • Because NAU's bins have matching specs, the design should be applicable there with little to no modification
  • Any differences in bin design mean starting from scratch on training data since the environment has changed.
  • A fully modular "drop-in" system that works in any bin is significantly harder and more expensive.

Recommendation

A pilot should target one known bin design to prove the concept before attempting modularity.

Decision Framework

Three Options

Option A

Engage Consultant — Stage-Gated Pilot

2-bin pilot at UMass Lowell · ~$36k–$65k all-in · 4–5 months

Start with known bin geometry. Decision gate after pilot: does it work well enough? Only expand to 9 bins if pilot succeeds.

+ Most thorough path; real AI/ML expertise; proves concept before scaling – Significant investment before we confirm the problem it solves is actually driving losses
Option B

Small Internal Pilot — No Consultant

Off-the-shelf cameras · basic motion detection · lower upfront cost

Use existing team knowledge to prototype a simpler version. Skip custom throat design to start.

+ Cheaper to start; keeps optionality; builds internal knowledge – Distraction from checkout station + Transact work; No AI/ML expertise on team; risk of getting nowhere or building something that doesn't actually validate the concept
Option C

Shelve for Now

Gather loss data first · revisit when loss driver is confirmed

Focus on the root causes suggested by NAU and Lowell feedback. Confirm return-station fraud is a significant driver before investing.

+ No investment risk; addresses "are we solving the right problem?" first – If Alison's instinct is right, we lose time; consultant may not be available later

Before We Decide

Key Unknowns & Recommended Next Steps

What we don't know yet

Whether return-station fraud is actually a significant source of container loss (NAU + Lowell evidence suggests maybe not)
Whether the no-illumination approach works in real dining hall lighting on wet, reflective surfaces
Whether NAU's existing bins can accommodate the system — or if they'd adopt USEFULL return stations
Whether Nadia's bandwidth can absorb this without impacting the roadmap
Exact mechanical fit — 1.75" rails are tight once fasteners, brackets, and cable bends are added

Regardless of which option — immediate next steps

  1. 1
    Quantify the actual loss rate at return stations specifically, vs. other points in the container lifecycle
  2. 2
    Clarify whether NAU needs to use existing bins or would adopt USEFULL return stations
  3. 3
    If proceeding: start with a 2-bin pilot at one site with known geometry — don't try to solve modularity first

A camera-based return verification system is technically plausible. The question is whether it solves the right problem, and whether the investment is justified given what we're learning about where containers are actually being lost.
